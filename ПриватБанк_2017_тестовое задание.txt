Тестовое задание


Разработать серверное и клиентское приложения, реализующие функциональность справочника по банковским вкладам.
1.           Для каждого вклада необходимо хранить следующую информацию:
—Name — название банка;
—Country — страна регистрации;
—Type — тип вклада (до востребования, срочный, расчетный, накопительный, сберегательный, металлический);
—Depositor — имя вкладчика;
—Account id — номер счета;
—Amount on deposit — сумма вклада;
—Profitability — годовой процент;
—Time constraints — срок вклада.
2.       Серверное приложение должно сохранять информацию в файле (файлах) с использованием механизма сериализации либо в формате XML или JSON.
3.       Взаимодействие сервера с клиентами должно происходить посредством соединения через сокет на определенном порту.
4.       Сервер должен поддерживать множественные подключения клиентов.
5.       Сервер должен по сети принимать команды и отвечать на них:
·         list – выдать список всех вкладов
·         sum – выдать общую сумму вкладов
·         count – выдать количество вкладов
·         info account <account id> - информация по счету
·         info depositor <depositor> - информация по имени вкладчика. Предполагается, что у одного вкладчика может быть несколько вкладов, поэтому в ответе должен быть список вкладов.
·         show type <type> - список всех вкладов указанного типа
·         show bank <name> - список всех вкладов в указанном банке
·         add <deposit info> - добавить информацию о вкладе (если успешно – в ответе «OK», если вклад с таким id уже есть или сумма параметры вклада недопустимы (сумма, процент, срок вклада ≤0, в ответ – сообщение об ошибке)
·         delete <account id>
6.       Формат передаваемой информации  выбрать самостоятельно.
7.       Клиентское приложение должно предоставить интерфейс для ввода команд и отображения ответов сервера
8.       Выполненное задание выложить на github.com, обязательно указав в файле readme авторство и способы коммуникации с Вами (Ваши e-mail, skype, телефон). Ссылку на Ваше решение в github прислать на e-mail
